# Contributors

Heron exists because of the people who build it, file issues, ship translations,
write docs, and share their job-search outcomes. This file lists everyone who's
contributed -- code or otherwise.

## Heritage

Heron is a hard fork of an open-source CLI-driven job-search system by
**santifer**. The original archetypes, scoring logic, negotiation scripts,
and proof-point structure are theirs; this fork extends them with multi-user
RBAC, a SvelteKit dashboard, Capacitor native apps, and an autonomous-apply
pipeline. See the [README's Acknowledgements section](README.md#acknowledgements)
for the case-study link.

## Maintainers

- **[kaelys-js](https://github.com/kaelys-js)** -- maintainer, fork lead

See [docs/GOVERNANCE.md](docs/GOVERNANCE.md) for the contributor ladder
(Participant -> Contributor -> Triager -> Reviewer -> Maintainer) and the
expectations at each level.

## All contributors

The list below is maintained by the [all-contributors](https://allcontributors.org)
bot. Comment `@all-contributors please add @username for code, docs, ideas`
on any open PR / issue to add someone (you can list multiple contribution types
per comment -- see the [emoji key](https://allcontributors.org/docs/en/emoji-key)).

Recognized contribution types include code, documentation, design, ideas,
maintenance, project-management, review, testing, translation, video, blog
posts, talks, bug reports, and showcases -- everything that makes a project
welcoming, not just commits.

## How to be listed here

- **Code / docs / fixes**: open a PR; if it merges, you're on the
  commit-history grid above automatically.
- **Non-code contributions** (translation, bug report, ideas, design,
  community help, etc.): leave a comment on any open issue or PR with
  `@all-contributors please add @your-handle for translation, ideas`
  -- the bot opens a PR adding you to the all-contributors list.
- **Showcase / "I got hired with Heron"**: open a thread in
  [Discussions](https://github.com/kaelys-js/heron/discussions) using the
  Showcase or I-got-hired template -- threads are linked from the README's
  community section.
