### Gap Analysis and Mitigation

1. **Python / FastAPI** — Hard or nice-to-have?  
   Likely a real production requirement — Parloa uses both languages in their platform. Cole's risk: may lose to candidates with active Python production experience.  
   Mitigation: Be direct — "TS/Node is my production language; Python is readable and I've used it for scripting. FastAPI's async model maps directly to Node async patterns. I'd be productive within a sprint." Don't fake proficiency; frame the transfer speed.

2. **Kubernetes** — Nice-to-have  
   Cole has Docker, CI/CD, cloud infra (AWS + GCP). K8s is adjacent. For a Staff engineer, infra-level K8s tuning is often delegated.  
   Mitigation: "Operational exposure via managed GKE/EKS — pods, services, deployments, rollout strategies. Haven't tuned scheduler internals."

3. **Azure** — Real but learnable  
   Parloa is German; Azure is common in German enterprise. Cole has AWS + GCP.  
   Mitigation: "Same cloud primitives, different vendor interface. The concepts are identical — the CLI and IAM model take a day, not a month."

4. **Kafka / NATS** — Significant gap  
   Cole has real-time data pipelines via Cloudflare Workers + Analytics Engine. Different pattern, not the same tech.  
   Mitigation: "I've shipped production real-time event streaming on a different substrate (Cloudflare Analytics Engine). The consumer/producer/topic model is something I've studied but haven't used in production. Honest gap."

5. **Org-level impact** — The biggest gap  
   The JD explicitly requires "org-level or industry-level impact." Cole has worked as a senior IC in companies up to ~50-person engineering team scale. The cross-cutting work at Enzuzo is real (cloud, CI/CD, analytics pipeline) but the org is smaller.  
   Mitigation: Frame Enzuzo correctly — "In a 15-person engineering team I owned the entire cloud infrastructure, the consent analytics system, and the CI/CD backbone. Everyone felt it when it was fast; everyone felt it when it broke. That's org-level by necessity."
