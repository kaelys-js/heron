### Análisis de gaps

#### 1. Golang — HARD BLOCKER
- **¿Bloqueante o nice-to-have?** Hard blocker. El JD dice "Proficiency in Golang" entre los requisitos principales. Este es un rol de sistemas CDN escrito en Go: routing daemon, TLS stack, proxy layer.
- **¿Experiencia adyacente?** No. TypeScript/Node.js es un runtime interpretado de alto nivel; Go es compilado, tipado estático, con gestión de concurrencia vía goroutines. Son paradigmas distintos. No es posible reclamar Go proficiency a partir de TS/Node.
- **¿Proyecto portfolio que cubra este gap?** No hay ningún proyecto Go en cv.md ni en article-digest.
- **Plan de mitigación:** Ninguno a corto plazo. Cole tendría que aprender Go activamente (meses de practice + proyectos) antes de poder aplicar crediblemente a este rol.

#### 2. Protocolos de red / CDN depth — Gap moderado-alto
- **¿Bloqueante?** Soft-to-hard. El rol requiere entender TLS termination, HTTP routing, anycast, caching headers a nivel de implementación, no de consumo.
- **¿Experiencia adyacente?** Cloudflare Workers en Enzuzo es adyacente (entiende el modelo conceptual de edge), pero Workers son una abstracción JS sobre la red — no el mismo nivel que implementar un CDN proxy.
- **Mitigación parcial:** Cloudflare Workers + Analytics Engine experience demuestra comprensión del paradigma edge, pero no reemplaza el conocimiento de red a nivel de sistemas.

#### 3. Kubernetes / Terraform — Soft gaps (preferred)
- K8s no en CV pero Docker en producción. Gap bridgeable a mediano plazo.
- Terraform no en CV. CI/CD automation experience es adyacente.

#### 4. Lua — Soft gap (preferred)
- Usado en nginx configs y CDN scripting. Cole no tiene experiencia documentada.
