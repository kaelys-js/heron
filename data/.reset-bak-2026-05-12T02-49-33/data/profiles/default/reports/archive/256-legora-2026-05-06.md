### Gaps — Mitigation Strategies

1. **Azure (HARD GAP):** Primary cloud mismatch. Cole has production AWS + GCP, not Azure. Mitigation: Frame AWS→GCP migration as proof of cross-cloud adaptability ("migrated production workloads across cloud providers"). Azure core concepts map well from GCP (IAM, compute, networking, blob storage). Short ramp time for an engineer who's already moved clouds. Not a blocker at startups where willingness to learn matters more than cert.

2. **Pulumi (SOFT GAP):** Legora uses Pulumi; Cole has Terraform. Same IaC mental model, different DSL. Mitigation: List "Terraform / Pulumi (IaC)" in skills. Cole's TypeScript fluency is actually an advantage — Pulumi's TS SDK is its most mature and most used.

3. **Kubernetes not explicit (SOFT GAP):** GKE at Enzuzo means production K8s, but "Kubernetes" itself isn't in cv.md. Mitigation: Add "Kubernetes" to Skills/Infra section; it's accurate given GKE use.

4. **Dedicated platform/SRE framing (FRAMING GAP):** Cole is primarily a software engineer who does infra — not an SRE/platform engineer by default. Mitigation: For a targeted application, reframe the Enzuzo section to lead with infra ownership scope (migration, CI/CD, edge pipeline infrastructure) rather than feature delivery. The substance is there; the framing isn't.
