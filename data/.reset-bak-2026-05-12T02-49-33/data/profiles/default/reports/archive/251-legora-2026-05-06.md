### Gap Analysis

1. **Python** — HARD-ish blocker. JD lists it first in required skills, alongside TypeScript. If Legora's backend is Python + TypeScript (dual-language, which is common in AI startups), Cole would need to onboard to Python. Mitigation: Node.js covers the same API/service scope; Python is a short ramp for scripting/tooling. In cover: "TypeScript/Node primary; I've worked in polyglot environments and Python scripting is a short onramp." Risk: medium — if their primary product backend is Python, this is a real gap.

2. **Azure** — Nice-to-have, not a blocker. Cole has AWS (production) + GCP (production, full migration). Azure concepts are transferable in weeks. Low risk.

3. **Kubernetes** — Nice-to-have. Cole has Docker + CI/CD depth. K8s basics are approachable with that foundation. Low risk.

4. **Elasticsearch** — Nice-to-have for most roles; legal search/document retrieval at Legora might weight it more. No mitigation other than honest acknowledgement. Low risk unless search is the core product feature.

5. **IaC** — Nice-to-have. Cole's CI/CD and infrastructure experience demonstrates the same ownership mindset. Low risk.

**Bottom line:** Strong core match (TypeScript + React + Node + PostgreSQL + CI/CD), with a real Python gap and cloud platform mismatch. For the right candidate posture, this is addressable.
